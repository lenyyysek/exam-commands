4. Docker (контейнеризация)
РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование ПО: Docker Engine (версия 24+)
Краткое описание
Docker — это платформа для разработки, доставки и запуска приложений в изолированных контейнерах. Контейнеры легче виртуальных машин, так как используют ядро хост-системы, но изолируют приложение и его зависимости. Docker позволяет упаковать приложение со всеми библиотеками и конфигурациями в образ, который затем можно запустить на любой системе с Docker. Это решает проблему «у меня работает, а на сервере нет». Docker широко используется в DevOps для автоматизации развертывания и масштабирования приложений. С Docker Compose можно описывать многоконтейнерные приложения (например, веб-сервер + база данных + кэш) в одном файле.
Запуск программы
Docker работает как фоновый сервис (демон). Основное взаимодействие — через команды в терминале:
•	Проверка версии: docker --version
•	Проверка статуса службы: sudo systemctl status docker
•	Запуск тестового контейнера: docker run hello-world
Запланированный выход
•	Остановка работающего контейнера: docker stop <имя_или_ид_контейнера>
•	Остановка всех контейнеров: docker stop $(docker ps -q)
•	Остановка фонового сервиса Docker: sudo systemctl stop docker (обычно не требуется)
Основные функции
1.	Создание и управление изолированными контейнерами с приложениями
2.	Сборка собственных образов на основе Dockerfile (автоматизированное описание окружения)
3.	Управление образами — скачивание готовых образов из репозиториев (Docker Hub), удаление, экспорт
4.	Работа с несколькими контейнерами одновременно с помощью Docker Compose
5.	Изоляция сетей, томов и ресурсов между контейнерами
6.	Масштабирование приложений — запуск множества экземпляров одного контейнера
7.	Интеграция с системами CI/CD для автоматической сборки и развертывания
Основные приемы работы
1. Проверка установки
Выполните: docker --version. Если отображается версия, Docker установлен. Для проверки работы запустите тестовый контейнер: docker run hello-world. Если появилось приветственное сообщение, всё работает.
2. Запуск контейнера из готового образа
Загрузите образ Ubuntu и запустите интерактивную сессию bash:
docker pull ubuntu
docker run -it ubuntu bash
Вы окажетесь внутри контейнера, где можно выполнять любые команды. Выход из контейнера: exit.
3. Просмотр запущенных контейнеров
•	Только работающие: docker ps
•	Все (включая остановленные): docker ps -a
4. Просмотр образов на локальной машине
docker images — список всех скачанных и собранных образов.
5. Остановка контейнера
Узнайте ID контейнера из docker ps, затем:
docker stop <id> — плавная остановка.
docker kill <id> — принудительная остановка.
6. Удаление контейнера
docker rm <id> — удаляет остановленный контейнер.
docker rm -f <id> — удаляет контейнер принудительно, даже если он работает.
7. Удаление образа
docker rmi <image_id> — удалить локальный образ.
8. Запуск контейнера с пробросом портов
Запустите веб-сервер на порту 80 внутри контейнера и пробросьте его на порт 8080 хоста:
docker run -d -p 8080:80 nginx
После этого в браузере по адресу http://localhost:8080 будет доступна страница Nginx.
9. Сборка образа из Dockerfile
Создайте файл Dockerfile:
FROM alpine:latest
RUN apk add --no-cache python3
COPY script.py /app/
CMD ["python3", "/app/script.py"]
Соберите образ: docker build -t myapp .
Запустите собранный контейнер: docker run myapp
10. Работа с томами (сохранение данных)
Запустите контейнер с примонтированной папкой хоста:
docker run -v /home/user/data:/data ubuntu
Теперь данные, сохраненные в /data внутри контейнера, будут доступны в /home/user/data на хост-машине.
11. Использование Docker Compose
Создайте файл docker-compose.yml:
version: '3'
services:
  web:
    image: nginx
    ports:
      - "8080:80"
  db:
    image: postgres
    environment:
      POSTGRES_PASSWORD: secret
Запустите оба контейнера одновременно: docker-compose up -d
Остановите: docker-compose down
12. Логи контейнера
docker logs <id> — показать логи работающего контейнера.
docker logs -f <id> — следить за логами в реальном времени.
13. Выполнение команды внутри работающего контейнера
docker exec -it <id> bash — зайти в контейнер в интерактивном режиме.
