3. Node.js (веб-разработка)
РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование ПО: Node.js (версия LTS)
Краткое описание
Node.js — это среда выполнения JavaScript на сервере, построенная на движке V8 (Google Chrome). Она позволяет писать серверные приложения на JavaScript, что особенно удобно для веб-разработчиков, которые хотят использовать один язык как на клиенте, так и на сервере. Node.js имеет асинхронную событийно-ориентированную архитектуру, что делает ее эффективной для создания высоконагруженных приложений реального времени (чаты, стриминг, API). Вместе с менеджером пакетов npm образует огромную экосистему библиотек и фреймворков.
Запуск программы
•	Проверка установки и версии: node -v
•	Запуск JavaScript-файла: node script.js
•	Интерактивный режим (REPL): введите node в терминале и нажимайте Enter после каждой команды
•	Выход из интерактивного режима: дважды нажать Ctrl+C или ввести .exit
Запланированный выход
•	Остановка выполняющегося скрипта: Ctrl+C
•	Завершение интерактивного режима: Ctrl+C дважды или .exit
•	Корректное завершение сервера: в коде приложения нужно использовать process.exit() или сигналы завершения
Основные функции
1.	Выполнение кода JavaScript вне браузера, непосредственно на сервере
2.	Асинхронная работа с файловой системой (чтение, запись, удаление файлов)
3.	Создание HTTP-серверов и обработка входящих запросов
4.	Работа с базами данных через драйверы (PostgreSQL, MongoDB, MySQL)
5.	Управление пакетами и зависимостями через npm (Node Package Manager)
6.	Создание REST API и веб-сокетов для обмена данными в реальном времени
7.	Обработка потоков данных (работа с большими файлами, стриминг)
Основные приемы работы
1. Проверка установки
В терминале выполните: node -v. Если отобразилась версия, значит Node.js установлен правильно.
2. Создание первого скрипта
Создайте файл app.js и напишите:
javascript
console.log('Hello, World!');
Запустите: node app.js. В терминале появится сообщение.
3. Инициализация нового проекта
В папке проекта выполните: npm init -y. Будет создан файл package.json с настройками по умолчанию. Этот файл хранит информацию о проекте и зависимостях.
4. Установка внешних пакетов
Для установки пакета (например, Express — популярный веб-фреймворк):
npm install express
Пакет будет загружен и сохранен в папку node_modules, а также добавлен в зависимости package.json.
5. Простой HTTP-сервер на Node.js
Создайте файл server.js:
const http = require('http');

const server = http.createServer((req, res) => {
    res.writeHead(200, {'Content-Type': 'text/plain'});
    res.end('Привет, мир!');
});

server.listen(3000, () => {
    console.log('Сервер запущен на порту 3000');
});
Запустите: node server.js. Откройте браузер и перейдите на http://localhost:3000 — вы увидите текст.
6. Установка пакета глобально
Некоторые пакеты удобно устанавливать глобально, чтобы использовать их из любого места системы:
sudo npm install -g nodemon
Nodemon автоматически перезапускает сервер при изменениях файлов, что удобно при разработке.
7. Использование nodemon
Запустите сервер с автоматической перезагрузкой:
nodemon server.js
Теперь при каждом сохранении файла сервер будет перезапускаться автоматически.
8. Установка и использование Express (более простой сервер)
npm install express
Файл app.js:
const express = require('express');
const app = express();

app.get('/', (req, res) => {
    res.send('Hello from Express!');
});

app.listen(3000, () => console.log('Сервер запущен'));
Запуск: node app.js. Сервер работает на порту 3000.
9. Удаление пакетов
npm uninstall express — удалить пакет и убрать его из зависимостей.
10. Обновление пакетов
npm update — обновить все пакеты до актуальных версий в рамках диапазонов, указанных в package.json.
11. Просмотр установленных пакетов
npm list — показать все установленные пакеты в текущем проекте (рекурсивно).
npm list --depth=0 — показать только корневые зависимости.
