2. PostgreSQL (база данных)
РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
Наименование ПО: PostgreSQL 16
Краткое описание
PostgreSQL — это объектно-реляционная система управления базами данных с открытым исходным кодом. Она считается одной из самых надежных и функциональных СУБД, поддерживает сложные запросы, транзакции, внешние ключи, триггеры и хранимые процедуры. PostgreSQL широко используется в веб-приложениях, аналитических системах и корпоративных проектах благодаря высокой производительности и соответствию стандартам SQL.
Запуск программы
•	Подключение к серверу БД от имени суперпользователя: sudo -u postgres psql
•	Подключение к конкретной базе: psql -d имя_базы -U имя_пользователя
•	Выход из оболочки psql: \q
Запланированный выход
•	В интерактивной оболочке psql: \q или Ctrl+D
•	В графических клиентах (pgAdmin, DBeaver): «Файл» → «Выход» или закрыть окно
Основные функции
1.	Создание, удаление и управление базами данных
2.	Создание таблиц с различными типами данных (числовые, строковые, дата/время, JSON, массивы)
3.	Выполнение SQL-запросов — SELECT, INSERT, UPDATE, DELETE
4.	Управление пользователями и правами доступа
5.	Создание индексов для ускорения поиска
6.	Резервное копирование и восстановление баз данных
7.	Поддержка транзакций (ACID) для надежности операций
Основные приемы работы
1. Подключение к серверу
Откройте терминал и выполните: sudo -u postgres psql. Вы окажетесь в интерактивной оболочке, где можно выполнять SQL-команды.
2. Просмотр существующих баз данных
В оболочке psql выполните: \l. Будут показаны все базы данных на сервере.
3. Создание новой базы данных
Выполните: CREATE DATABASE mydb; — создаст базу с именем mydb.
4. Подключение к базе данных
\c mydb — переключиться на базу mydb. После этого все команды будут выполняться в контексте этой базы.
5. Создание таблицы
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT NOW()
);
Создается таблица users с полями: id (автоинкремент), name (обязательное текстовое поле), email (уникальное), created_at (дата создания по умолчанию).
6. Вставка данных
INSERT INTO users (name, email) VALUES ('Иван Петров', 'ivan@mail.ru');
INSERT INTO users (name, email) VALUES ('Мария Смирнова', 'maria@mail.ru');
7. Просмотр данных
SELECT * FROM users;
SELECT name, email FROM users WHERE name LIKE 'И%';
8. Просмотр структуры таблицы
\d users — показать структуру таблицы users (поля, типы, индексы).
9. Просмотр всех таблиц в базе
\dt — показать список всех таблиц в текущей базе.
10. Удаление данных
DELETE FROM users WHERE id = 1;
11. Обновление данных
UPDATE users SET email = 'new@mail.ru' WHERE name = 'Иван Петров';
12. Создание пользователя
CREATE USER myuser WITH PASSWORD 'mypass';
GRANT ALL PRIVILEGES ON DATABASE mydb TO myuser;
13. Выход из оболочки
\q или Ctrl+D.
